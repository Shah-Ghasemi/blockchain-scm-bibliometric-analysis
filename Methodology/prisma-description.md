PRISMA-Based Data Collection Methodology

This study adopts the PRISMA framework as a transparent reporting guideline to document the process of identifying, screening, and selecting relevant academic articles.

The framework is used exclusively to enhance:

- Transparency in data collection
- Reproducibility of the research process
- Consistency in screening and filtering procedures

Data Collection Process

The bibliographic data was collected from two major academic databases:

- Scopus
- Web of Science

The workflow includes the following steps:

1. Definition of search strategy and keywords
2. Identification of relevant records
3. Screening based on inclusion criteria
4. Removal of duplicate records
5. Final dataset construction for analysis

Notes on Methodological Use

It is important to note that the PRISMA framework is used solely for documenting the data selection workflow and not as a full systematic review protocol.

The primary objective is to ensure a structured and transparent bibliometric data collection process.

Tools and Implementation

The data integration and preprocessing were conducted using:

- R programming environment
- Bibliometrix / Biblioshiny package

These tools support efficient data merging, cleaning, and preparation for bibliometric analysis.
